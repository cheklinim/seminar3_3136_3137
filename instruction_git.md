# **Инструкция по работе с системой контроля версий Git**

![Эмблема Git](git.jpg)

Git (произносится «гит») — распределённая система управления версиями. Проект был создан Линусом Торвальдсом для управления разработкой ядра Linux, первая версия выпущена 7 апреля 2005 года. На сегодняшний день его поддерживает Джунио Хамано.

## Инициализация репозитория

Чтобы создать (инициализировать) новый репозиторий нужно в терминале ввести команду:

    git init

Репозиторий будет создан в той папке, из которой вызывалась команда

## Проверка состояния репозитория

Чтобы проверить текущее состояние репозитория нужно ввести в терминале команду:

    git status

## Добавление изменения к отслеживанию версионности

Чтобы добавить сделанное изменение к отслеживанию (поместить в индекс) нужно ввести команду:

    git add <имя файла>

где вместо <имя файла> вводится путь к файлу относительно расположения репозитория.

## Фиксация изменений

Чтобы зафиксировать изменение используется команда:

    git commit

В таком случае откроется окно для ввоба краткого описания сделанных изменений.

Чтобы сделать это одновременно с фиксацией используется команда:

    git commit -m "комментарий"

## Просмотр истории изменений

Чтобы посмотреть историю изменений используется комада

    git log

Для просмотра изменений с выводом одного коммита в одну строку используется команда

    git log --oneline

Для просмотра всех имеющихся коммитов используется команда

    git log --all

Для просмотра лога с графическим изображением веток используется команда

    git log --graph

Все указанные флаги могут использоваться вместе:

    git log --all --oneline --graph

## Просмотр различий между изменениями

Для просмотра отличий между текущим состоянием репозитория и последним сохраненным изменением используется команда

    git diff

Можно также посмотреть разницу между любыми двуми коммитами. Для этого используется команда

    git diff <хэш1> <хэш2>

## Переключение на нужное изменение

Чтобы переключиться на нужный коммит используется команда

    git checkout <хэш>

## Ветки в Git

### Создание новой ветки

Чтобы создать новую ветку используется команда

    git branch <имя_ветки>

### Просмотр всех веток

Чтобы посмотреть какие ветки существуют и на какой мы находимся используется команда

    git branch

### Переключение между ветками

Чтобы переключиться на другую ветку используется команда

    git checkout <имя_ветки>

### Слияние веток

Чтобы влить одну ветку в другую необходимо находясь в целевой ветке (КУДА будем делать слияние) выполнить команду

    git merge <имя_вливаемой_ветки>

### Конфликты при слиянии

Если одна и та же строка в разный версиях записана по разному возникнет конфликт.
Чистый гит автоматически сохраняет оба изменения, далее требуется вручную внести нужные правки и сделать коммит.

VSСode дает возможность выбрать какое изменение сохранить (входящее, существующее или оба).

### Удаление ветки

Чтобы удалить ветку, которая больше не нужно (например после слияния) используется команда

    git branch -d <имя_ветки>



## Создание нового репозитория на GitHub

* Создать новый удаленный репозиторий можно нажав кнопку "+" на главной странице профиля.

* Репозиторий может быть публичный и непубличный, это нужно указать при создании. В публичном репозитории все файлы будут в открытом доступе всем. 

## Связывание удаленного и локального репозитория

* После создания нового репозитория на странице Github нужно связать удаленный репозиторий с локальным. Для этого нужно скопировать ссылку, указанную в подсказке на странице github и вставить ее в терминал в нужном локальном репозитории. Также нужно проконтролировать название главное ветки и при необходимости его изменить, по умолчанию ветка будет названа main.

## Отправка коммитов в удаленный репозиторий

* Чтобы отправить изменения в удаленный репозиторий используется команда 

    git push

Делается после того, как все изменения отправлены в коммит. Сделать push можно только в СВОЙ репозиторий! 

## Получение коммитов из удаленного репозитория

* Изменения в файл можно вносить напрямую на Github. После этого их нужно скачать в локальный репозиторий, используя команду:

    git pull

## Копирование чужого репозитория

* Скопировать информацию с чужого репозитория можно командой 

    git clone ссылка

* Если при помощи этой команды скопировать данные, внести изменения и сделать push - будет ошибка, т.к. данные будут пытаться отправиться в чужой репозиторий. 

## Работа с чужими репозиториями, совместная работа

* При совместной работе сначала всегда надо актуализировать данные на своем локальном репозитории, использовав git pull.

* Чтобы работать с чужими репозиториями, вносить туда предполагаемые изменения, используется такой порядок команд:

1. Зайти на необходимый репозиторий. Нажать там FORK. Эта команда копирует данные чужого репозитория в мой репозиторий. При этом сохраняется информация, откуда взяли эти данные. 
2. Сохранить скопированные данные на свой новый удаленный репозиторий, присвоить ему имя и настройки. 
3. Оттуда взять ссылку для копирования информации в локальный репозиторий. 
4. В новую папку на своем локальном репозитории в терминале создать команду

    git clone ссылка

5. Создать новую ветку, чтобы не пушить потом в главную ветку чужого репозитория. 
6. После того, как сделаны нужные изменения, сделать коммит и выгрузить данные в свой репозиторий командой

    git push git push --set-upstream название ветки

    Такой вариант команды используется потому, что мы пушим ветку, которой нет на удаленном репозитории. На github при этом добавиться новая ветка. 

7. После этого на github появится кнопка 

    COMPARE/PULL REQUEST

8. Нажать на данную кнопку, внести комментарии при необходимости и нажать CREATE.
